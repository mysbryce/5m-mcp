agent_apiAn MCP server for FiveM

Let any MCP-aware agent develop, debug, and live-test FiveM resources against a running server.

Quick start
View on GitHub

One drop-in FiveM resource

No external sidecar. Drop the folder into resources/[agent]/, ensure agent_api, paste the printed MCP config into your client. Ready.

36+ tools out of the box

Filesystem (sandbox-guarded), resource lifecycle, console capture, live-player probes, dynamic client + server natives, allowlisted shell, NUI screenshots.

Framework bridges

ESX, ox_lib, oxmysql plugins auto-detect on boot. Reflective dispatchers expose every method of each framework with a readonly verb gate.

Speaks real MCP

HTTP transport for Claude Code, Cursor, and any spec-compliant client. Bundled stdio shim for clients that need stdin/stdout.

Grill workflow for scaffolding

scaffold_fivem_resource_workflow tool + matching MCP prompt force a structured Q&A before any file is written. Vite UI, ox_lib modules, config split — all asked, never assumed.

Built for safety on a live server

Token bootstrap, per-resource locks, readonly mode, blocklists for dangerous natives, ACE-aware lifecycle, audit log of every action. Self-targeting lifecycle commands are hard-refused.
